# Runtime Artifact Model V2

This document defines a universal, unified execution artifact database schema to replace the current CLI-centric logging structure in Nexus. It details the schema, subtypes, and data representation strategy.

---

## 1. Schema Strategy: Unified Model with JSON Meta-payload

Rather than fragmenting the database with multiple tables for CLI, Agent, and Research output models (which complicates indexing and querying), we adopt a **Unified Entity with Polymorphic JSON metadata** pattern.

Under [models.py](file:///D:/nexus/nexus/memory/models.py), the `ExecutionArtifactRecord` table schema remains clean and generic, while execution-specific primitives are stored inside the `data` JSON payload.

```
                  +-----------------------------------+
                  |     ExecutionArtifactRecord       |
                  +-----------------------------------+
                  | - id: UUID                        |
                  | - execution_id: UUID (FK)         |
                  | - artifact_type: String           |
                  | - name: String                    |
                  | - content: Text                   |
                  | - data: JSON                      |
                  +-----------------+-----------------+
                                    |
            +-----------------------+-----------------------+
            |                       |                       |
            v                       v                       v
    [ CLI Artifacts ]      [ Agent Artifacts ]     [ Research Artifacts ]
    - stdout.log           - plan.json             - citations.json
    - stderr.log           - trajectory.json       - findings.json
    - changes.diff         - file_patch.diff       - research_brief.md
```

---

## 2. Artifact Mapping by Runtime Category

### A. CLI Runtime Artifacts
* **`stdout`**:
  * `name`: `"stdout.log"`
  * `content`: Raw standard output character logs.
  * `data`: `{"size_bytes": 1024, "exit_code": 0}`
* **`stderr`**:
  * `name`: `"stderr.log"`
  * `content`: Raw standard error logs or traceback details.
  * `data`: `{"size_bytes": 240}`
* **`diff`**:
  * `name`: `"changes.diff"`
  * `content`: Unified diff output from `git diff`.
  * `data`: `{"modified_files": ["auth.py"]}`

### B. Agent Runtime Artifacts
* **`agent_plan`**:
  * `name`: `"plan.json"`
  * `content`: Serialized task steps compiled by the planner.
  * `data`: `{"total_steps": 5, "completed_steps": 4}`
* **`agent_trajectory`**:
  * `name`: `"trajectory.json"`
  * `content`: JSON array containing thoughts, reasoning steps, tool calls, and tool outputs.
  * `data`:
    ```json
    {
      "reasoning_steps_count": 8,
      "tool_invocations": {
        "read_file": 3,
        "write_file": 1
      }
    }
    ```
* **`diff`**:
  * `name`: `"changes.diff"` (or individual patches)
  * `content`: Filesystem diff generated by agent tool edits.
  * `data`: `{"modified_files": ["main.py"]}`

### C. Research Runtime Artifacts
* **`research_citations`**:
  * `name`: `"citations.json"`
  * `content`: Serialized array of search engine results, articles, and documentation URLs.
  * `data`: `{"total_sources_found": 12}`
* **`research_findings`**:
  * `name`: `"findings.json"`
  * `content`: JSON array of factoids and text clippings gathered.
  * `data`: `{"facts_count": 42}`
* **`research_report`**:
  * `name`: `"research_brief.md"`
  * `content`: Synthesized markdown report written by the LLM summary agent.
  * `data`: `{"sections": ["Summary", "Technical Analysis", "Recommendations"]}`

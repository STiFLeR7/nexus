# Executive Summary

**Why v1.0.1 existed.** The accepted v1.0.0 onboarding audit found a strong governed-execution kernel
wrapped in operational and honesty gaps: a fail-open authorization default, a silent execution-timeout
bug, a *documented-but-unbuilt* scheduler, documentation that described a pre-build project, a
simulated Nexus runtime, and a sandbox that executed on the host by default. v1.0.1 was scoped as a
**correctness, safety, and operational-completeness release — not a feature release.** Every change had
to trace to an accepted finding; the blueprint had to stay synchronized; every change required
validation and an ADR/report.

**What was discovered.** All six findings were validated first-hand (AP-101) and confirmed real. Two
were latent safety defects (A-001, A-002); one was a structural autonomy gap (A-003); one was
governance-rule self-violation — drift (A-004); two were honesty gaps where advertised capability
exceeded reality (A-005 Nexus, A-006 sandbox).

**What was fixed.** A-001 (fail-closed owner auth, startup + engine), A-002 (timeout correctness honoring
ADR-010 + `hard_limit`), and A-003 (a real single-node APScheduler foundation with six audited jobs) were
implemented under strict TDD. A-004 realigned all primary documentation to reality with a single
status source of truth.

**What was clarified.** A-005 and A-006 produced evidence-pinned classifications — Nexus is a
**Prototype**, the sandbox is **Unsafe By Default** — with full capability ledgers, gap analyses, risk
registers, and ADRs, **without** changing code (per their audit-only mandate).

**Net effect.** Nexus is now *honest, aligned, and operationally complete for single-node attended-to-
lightly-autonomous operation*. The core is safer (fail-closed, correct timeouts), the autonomy engines
are now actually triggered (scheduler), the documentation tells the truth, and the two remaining soft
spots (Nexus, sandbox) are precisely understood and bounded rather than overstated.

# Findings Closed

### A-001 — Fail-open owner authentication
- **Original finding:** Empty `owner_ids` silently disabled approval authorization; anyone could
  authorize governed execution (`config.py:42`, `approvals/service.py:94`, `bot.py:53`).
- **Resolution (AP-102):** Fail-closed at two layers — startup refuses to boot without owners; the
  approval engine refuses authorization when owners are unset or the actor is not an owner.
- **Evidence:** `api.py:67-82` (`_validate_startup_configuration`), `approvals/service.py:96-102`;
  tests `test_owner_auth_hardening.py` (4), `test_startup_validation.py` (2).
- **Current status:** ✅ **Resolved (implemented + tested).**

### A-002 — Execution timeout mismatch
- **Original finding:** Runners read a non-existent `research_timeout_seconds` field and silently used a
  300 s fallback, ignoring the ADR-010 tiers (`runners/claude.py:83` vs `config.py:83`).
- **Resolution (AP-102):** `resolve_execution_timeout(...)` reads the correct per-runtime field and
  **clamps to `execution.hard_limit`** so the ADR-010 ceiling cannot be exceeded.
- **Evidence:** `runners/base.py:9-30`; `claude.py:81` (`claude_timeout`), `gemini.py:86`
  (`gemini_timeout`), `nexus.py:119` (`research_timeout`); tests `test_timeout_resolution.py` (10).
- **Current status:** ✅ **Resolved (implemented + tested).**

### A-003 — Missing scheduler layer
- **Original finding:** APScheduler was a dependency but unused; research/briefing/metrics/approval-expiry
  engines were built but never autonomously triggered (`api.py:114-127`).
- **Resolution (AP-103B):** A real single-node APScheduler foundation behind a replaceable
  `SchedulerPort`, with six audited jobs — research, briefing, approval-expiry, metrics-aggregation,
  read-only outbox-health, read-only checkpoint-health — and four scheduler audit events.
- **Evidence:** `scheduling/scheduler.py`, `scheduling/jobs.py`, `gateway/outbox_health.py`,
  `memory/checkpoint_health.py`, `api.py` lifespan wiring; 17 tests; operational smoke (all six jobs
  fired with correct next-run times in Asia/Kolkata).
- **Current status:** ✅ **Resolved (implemented + validated, single-node).**

### A-004 — Documentation drift
- **Original finding:** README/STATUS/ROADMAP/CHANGELOG described a pre-build project; the blueprint
  violated its own "authoritative memory" rule.
- **Resolution (AP-104):** Realigned README, `blueprint/STATUS.md`, `blueprint/ROADMAP.md`,
  `CHANGELOG.md`, and the blueprint landing page; created the authoritative
  `architecture-status-summary.md` plus drift/state/history artifacts; logged residual *code-scoped*
  debt instead of editing code.
- **Evidence:** the AP-104 deliverable set; `NEXUS_DOCUMENTATION_ALIGNMENT_SUMMARY.md`.
- **Current status:** ✅ **Resolved (documentation aligned; accuracy ~3.0→9.0/10).**

### A-005 — Nexus runtime validation
- **Original finding:** Nexus contained simulated behaviors and placeholder logic.
- **Resolution (AP-105, audit-only):** Full capability ledger + trace + gap analysis; verdict
  **Prototype** (Concept Demonstration in default config). Confirmed: AsyncMock in prod path, hardcoded
  decorative plan, simulated `web_search`, no-op/uninvoked `terminate()`, no resume, always-`0` exit code.
- **Evidence:** `nexus-reality-audit.md`, `nexus-capability-ledger.md`,
  `nexus-execution-trace-analysis.md`, `nexus-gap-analysis.md`, `ADR-nexus-reality-audit.md`.
- **Current status:** ✅ **Resolved as a finding (reality established).** Remediation = future AP (not in scope).

### A-006 — Sandbox safety review
- **Original finding:** Default sandbox may execute on host; unknown provider may fall back to host;
  isolation not enforced by default.
- **Resolution (A-006, audit-only):** Full ledger + execution-path + risk + boundary analysis; verdict
  **Unsafe By Default**. Confirmed: `enabled=False` → host shell for all runtimes; policy decorative
  under Local; unknown provider fails open; Docker failures fail closed; audit complete.
- **Evidence:** `sandbox-safety-review.md`, `sandbox-capability-ledger.md`,
  `sandbox-execution-path-analysis.md`, `sandbox-risk-register.md`, `ADR-sandbox-safety-review.md`.
- **Current status:** ✅ **Resolved as a finding (reality established).** Remediation = future AP (not in scope).

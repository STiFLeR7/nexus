## Highlights

- 🔒 **Owner authorization now fails closed.** Nexus refuses to start without configured owners, and the
  approval engine rejects unauthorized actors. No more silent fail-open.
- ⏱️ **Execution timeouts are correct.** Runners honor the ADR-010 per-runtime tiers and are clamped to a
  hard limit (was: a silent 300 s fallback for every runtime).
- 🗓️ **The scheduler exists and runs.** A single-node APScheduler foundation now autonomously drives
  research, daily briefings, approval-expiration sweeps, metrics aggregation, and read-only health
  monitoring — every run audited.
- 📚 **Documentation tells the truth.** README, STATUS, ROADMAP, and CHANGELOG were realigned to the real
  v1.0.0/v1.0.1 system, with a single authoritative subsystem-status source.
- 🔎 **Two honesty audits delivered.** Hermes is formally a **Prototype**; the sandbox is **Unsafe By
  Default** — both fully evidenced, with no overstatement remaining in project docs.

## Operator upgrade notes

- **You must set `discord.owner_ids` (or `DISCORD_OWNERS`)** — Nexus will not start without it (A-001).
- **Scheduler is on by default** (`scheduling.enabled=true`); research **skips** until you configure
  `research_feeds`. Default timezone **Asia/Kolkata**; briefing at 08:00. Run **exactly one** instance
  (single-node; no cross-process lease yet).
- **Sandbox is OFF by default** — commands execute on the host. For isolation set `sandbox.enabled=true`,
  `sandbox.provider=docker` (Docker required), ideally `filesystem_policy=readonly`. Treat default
  deployments as host-executing (A-006).
- **Hermes is a Prototype** — do not route real "research"/autonomy tasks to `runtime_id="hermes"`
  expecting genuine findings (A-005).

## Known limitations (see `v1.0.1-risk-register.md`)

In-code version string still reads `0.1.0`; `/api/v1/status` reports `"stub"`; health is a boot-time
boolean; Gemini/Claude runtimes are generic shell runners; command guard is a bypassable substring
blacklist; Alembic migrations incomplete (`create_all` is the schema source); scheduler is single-node.
